## 2. 기본 사용법

### 2.1 API 요청 보내기

**① HTTP 메서드 선택**

URL 입력창 왼쪽 드롭다운에서 메서드를 선택합니다.

| 메서드 | 색상 | 주요 용도 |
|--------|------|-----------|
| `GET` | 🟢 초록 | 데이터 조회 |
| `POST` | 🔵 파랑 | 데이터 생성 |
| `PUT` | 🟡 노랑 | 데이터 전체 수정 |
| `DELETE` | 🔴 빨강 | 데이터 삭제 |
| `PATCH` | 🟠 주황 | 데이터 일부 수정 |
| `HEAD` | 회색 | 헤더만 조회 |
| `OPTIONS` | 회색 | 허용 메서드 확인 |

**② URL 입력**

```
https://api.example.com/users/1
```
- 반드시 `http://` 또는 `https://`로 시작해야 합니다.

**③ 📤 Send 버튼 클릭**

---

### 2.2 Query Parameters 추가

`📋 Params` 탭에서 키-값 쌍을 입력합니다.

| Key | Value |
|-----|-------|
| `page` | `1` |
| `limit` | `20` |

→ URL에 자동으로 `?page=1&limit=20` 이 붙어서 전송됩니다.

- **＋ 행 추가**: 새 파라미터 행 추가
- **✕**: 해당 행 삭제

---

### 2.3 Headers 설정

`🔧 Headers` 탭에서 요청 헤더를 입력합니다.

| Key | Value |
|-----|-------|
| `Authorization` | `Bearer eyJhbGci...` |
| `Accept` | `application/json` |

> 💡 **Content-Type**: Body(JSON) 사용 시 자동으로 `application/json`이 설정됩니다.

---

### 2.4 Body (JSON) 작성

`📄 Body (JSON)` 탭에서 JSON 형식으로 요청 본문을 입력합니다.

```json
{
  "name": "홍길동",
  "email": "hong@example.com",
  "age": 30
}
```

> ⚠️ `GET`, `HEAD`, `OPTIONS` 메서드는 Body를 사용하지 않으며, 해당 탭은 비활성화됩니다.

## 3. 응답 확인

요청 전송 후 화면 하단에 응답 패널이 표시됩니다.

### 3.1 메타 정보

| 항목 | 설명 |
|------|------|
| **Status** | HTTP 상태 코드 (색상 배지로 표시) |
| **Time** | 서버 응답 시간 (밀리초 또는 초) |
| **Size** | 응답 본문 크기 (B / KB / MB) |

**상태 코드 색상 의미**

| 범위 | 색상 | 의미 |
|------|------|------|
| 2xx | 🟢 초록 | 성공 |
| 3xx | 🟡 노랑 | 리다이렉트 |
| 4xx | 🟠 주황 | 클라이언트 오류 |
| 5xx | 🔴 빨강 | 서버 오류 |

### 3.2 Body 탭

- **ace 에디터**로 JSON 응답을 문법 강조(syntax highlight) 표시
- `📋 Raw 텍스트로 보기 (복사용)` 클릭 시 `st.code` 블록으로도 확인 가능

### 3.3 Headers 탭

응답 헤더를 표 형식으로 확인할 수 있습니다.

## 4. 히스토리 (좌측 사이드바)

### 4.1 히스토리 목록 확인

- 최근 요청 최대 **50건**이 자동 저장됩니다.
- 각 항목에 메서드·상태 코드·URL이 표시됩니다.

### 4.2 이전 요청 불러오기

`불러오기` 버튼 클릭 → 해당 요청의 URL, 메서드, Params, Headers, Body가 복원됩니다.

### 4.3 히스토리 전체 삭제

우측 상단의 `🗑️` 버튼 클릭

---

## 5. 에러 처리

요청 실패 시 에러 메시지와 해결 힌트가 표시됩니다.

| 에러 종류 | 표시 메시지 예시 | 힌트 |
|-----------|----------------|------|
| 잘못된 URL | "URL은 http:// 또는 https://로 시작해야 합니다." | URL scheme 확인 |
| 연결 오류 | "DNS 조회 실패 – 도메인을 확인하세요." | 도메인·인터넷 연결 확인 |
| 타임아웃 | "30초 이내에 서버가 응답하지 않았습니다." | 서버 상태·방화벽 확인 |
| SSL 오류 | "인증서가 유효하지 않거나 만료되었습니다." | 인증서 상태 확인 |

## 6. 사용 예시

### 예시 1: REST API 조회 (GET)

1. Method: `GET`
2. URL: `https://jsonplaceholder.typicode.com/posts/1`
3. 📤 Send 클릭
4. 응답: `200 OK` + JSON body 확인

### 예시 2: 데이터 생성 (POST with JSON)

1. Method: `POST`
2. URL: `https://jsonplaceholder.typicode.com/posts`
3. `📄 Body` 탭에 입력:
   ```json
   {
     "title": "테스트 포스트",
     "body": "내용입니다.",
     "userId": 1
   }
   ```
4. 📤 Send 클릭
5. 응답: `201 Created` + 생성된 리소스 확인

### 예시 3: 인증이 필요한 API

1. `🔧 Headers` 탭 → `Authorization` / `Bearer YOUR_TOKEN` 입력
2. URL 설정 후 📤 Send 클릭

| 상황 | 팁 |

| Body 초기화 | 히스토리 항목 `불러오기` → 새 요청으로 에디터 리셋됨 |
| 긴 URL 처리 | Params 탭에 파라미터 분리 입력 권장 |
